# ❤️ Heart Disease Prediction Dashboard

A polished, dashboard-style **Streamlit** web application that predicts the
likelihood of heart disease from patient clinical data, powered by a
pre-trained **Random Forest Classifier**.

## 📸 Overview

This project turns a trained ML model into an interactive, portfolio-ready
dashboard rather than a plain input form. It includes KPI stat cards, a
grouped patient-information form, a live patient summary panel, and an
animated probability meter for the prediction result.

## ✨ Features

- **Dashboard layout** — KPI strip, grouped input cards, and a live summary panel instead of a single long form.
- **Real-time patient summary** — updates as you fill in the form.
- **Risk prediction** — Low Risk (green) / High Risk (red) result cards.
- **Probability meter** — visual progress bar + percentage from `predict_proba()`.
- **Reset button** — instantly restores all fields to their defaults.
- **Loading spinner** — visual feedback while the model runs inference.
- **Expandable glossary** — explains every clinical feature in plain language.
- **Sidebar project info** — model type, accuracy, F1 score, and dataset details.
- **Responsive, modern medical theme** — white background, blue accent (`#2563EB`), rounded cards, soft shadows.

## 🗂️ Project Structure

```
heart-disease-prediction/
│
├── app.py               # Main Streamlit application
├── requirements.txt     # Python dependencies
├── README.md            # Project documentation
├── rf_heart.pkl         # Trained Random Forest model
├── scaler.pkl           # StandardScaler fitted on training data
└── column.pkl           # Ordered list of feature column names
```

> **Note:** `rf_heart.pkl`, `scaler.pkl`, and `column.pkl` must sit in the
> same folder as `app.py`. These are loaded with `joblib.load()`.

## ⚙️ How the Prediction Pipeline Works

1. The user fills in patient details through the dashboard form.
2. Categorical fields are one-hot encoded **exactly** the way they were
   during training (matching the reference/dropped category per feature).
3. A single-row `DataFrame` is built using the **exact column order**
   loaded from `column.pkl` — nothing is hardcoded.
4. Only the original numeric columns (`Age`, `RestingBP`, `Cholesterol`,
   `MaxHR`, `Oldpeak`) are transformed using the trained `scaler.pkl`.
5. The scaled feature row is passed to `rf_heart.pkl` for both
   `.predict()` and `.predict_proba()`.
6. The result and probability are rendered as an interactive dashboard card.

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/your-username/heart-disease-prediction.git
cd heart-disease-prediction
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv venv
source venv/bin/activate      # On Windows: venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add the model files

Place `rf_heart.pkl`, `scaler.pkl`, and `column.pkl` in the project root
(same folder as `app.py`).

### 5. Run the app

```bash
streamlit run app.py
```

The app will open automatically at `http://localhost:8501`.
